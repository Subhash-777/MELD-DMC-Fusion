## What This Project Does

Given a **TV dialogue** (e.g., a scene from *Friends*), the model reads each line spoken
and predicts the **emotion** behind it — choosing from:

| Label | Emotion | % in MELD Train |
|-------|---------|-----------------|
| 0 | neutral  | ~46% |
| 1 | surprise | ~12% |
| 2 | fear     | ~2%  |
| 3 | sadness  | ~7%  |
| 4 | joy      | ~13% |
| 5 | disgust  | ~2%  |
| 6 | anger    | ~11% |

The challenge: **class imbalance** (neutral dominates), short utterances lack context,
and emotions depend on *who said what before*. DMC-Fusion solves this by fusing all
three signals — what was *said* (text), *how* it was said (audio), and *facial expressions*
(visual) — using 5 turns of dialogue context.

## Dataset — MELD

**MELD** = Multimodal EmotionLines Dataset

| Property | Value |
|----------|-------|
| Source | TV show *Friends* (all 10 seasons) |
| Format | Video clips + transcripts + emotion labels |
| Splits | Train: 9,989 utterances / Dev: 1,109 / Test: 2,610 |
| Modalities | Text (transcript), Audio (WAV), Video (MP4 frames) |
| Classes | 7 emotions (neutral, surprise, fear, sadness, joy, disgust, anger) |
| Challenge | Severe class imbalance — neutral = 46%, fear = 2% |
| Paper | Poria et al., 2018 |

**Why MELD?**
- Real-world conversational data (not acted lab speech)
- Multimodal — all 3 modalities exist natively
- Industry benchmark — many published models report WF1 on this dataset
- Hard — sarcasm, cross-speaker context, code-switching between emotions

## Pretrained Models Used

### 1. RoBERTa-base (Text)
- **What it is:** A transformer language model with 125M parameters, pretrained on 160 GB of text
- **Why used:** Understands sentence meaning, sarcasm, negation — far better than word embeddings
- **Output:** 768-dimensional vector per token
- **Key config:** `bert_model = roberta-base`, `max_text_len = 192`
- **Fine-tuning strategy:** Frozen for first 5 epochs → then unfrozen at `lr = 2e-5`

### 2. WavLM-base+ (Audio)
- **What it is:** A self-supervised speech model trained on 94,000 hours of audio
- **Why used:** Captures pitch, tone, speaking rate, and prosody — signals RoBERTa never sees
- **Output:** Frame-level audio embeddings
- **Key config:** `wav2vec_model = microsoft/wavlm-base-plus`

### 3. EfficientNet-B4 (Visual)
- **What it is:** A CNN image classifier, pretrained on ImageNet
- **Why used:** Extracts facial expression features from video frames per utterance
- **Output:** 768-dim projection (`vis_proj_dim = 768`) matching text/audio hidden size
- **Key config:** `vis_backbone = efficientnet_b4`

## System Architecture

The model has **5 stages** that transform raw dialogue into emotion predictions:

```
RAW DIALOGUE (5 utterances)
        │
        ├─────────────────────┬──────────────────────┐
        ▼                     ▼                      ▼
[RoBERTa-base]         [WavLM-base+]         [EfficientNet-B4]
 Text Encoder           Audio Encoder          Visual Encoder
 (768-dim)              (768-dim)              (768-dim proj)
        │                     │                      │
        ▼                     ▼                      ▼
 [Text GRU]            [Audio GRU]            [Visual GRU]
 Context Encoder       Context Encoder        Context Encoder
 2-layer BiGRU         2-layer BiGRU          2-layer BiGRU
        │                     │                      │
        └─────────────────────┴──────────────────────┘
                              │
                   [Confidence Gating]
                   Soft weight per modality
                   (entropy regularised)
                              │
                   [Cross-Modal Attention]
                   8 heads × 2 layers
                   Each modality attends to others
                              │
                   [Fused Representation]
                        768-dim vector
                         /           \
              [Classifier Head]   [Shift Head]
              768 → 7 emotions    "Did emotion change?"
```

### Why each component?

| Component | Why it's there |
|-----------|----------------|
| **5-turn context** | Emotions build across conversation — 1 utterance alone is ambiguous |
| **Separate GRUs per modality** | Each modality has different temporal patterns — audio changes faster than text sentiment |
| **Confidence gating** | Dark scenes = poor visual; silent scenes = poor audio — gates auto-suppress weak modalities |
| **Cross-modal attention** | Learns *correlations* between modalities (e.g., sad face + flat tone + ambiguous words = sadness) |
| **Shift head** | Forces model to track emotion transitions, not just static labels per utterance |

## How Training Works

### Phase 1 — Fusion Warm-up (Epochs 1–5)
- RoBERTa is **frozen** — its 125M weights do not update
- Only the GRU encoders, attention, and classifier learn
- Purpose: Let the fusion layers find a good starting point before expensive backprop through RoBERTa
- Typical val WF1: 0.24–0.26 (model barely works — only neutral detected)

### Phase 2 — Full Fine-tuning (Epoch 6 onwards)
- RoBERTa is **unfrozen** at a tiny learning rate (`lr_pretrained = 2e-5`)
- All layers train simultaneously with different LRs:
  - Fusion layers: `lr_fusion = 1e-4` (10× larger — they need aggressive learning)
  - RoBERTa: `lr_pretrained = 2e-5` (tiny — preserve pre-trained language knowledge)
- Big jump expected at epoch 7–8 (val WF1 typically leaps from 0.30 → 0.53)

### Gradient Accumulation
- `batch_size = 2` (full dialogues per GPU step — VRAM constrained at 4GB)
- `grad_accum = 8` → effective batch = **2 × 8 = 16 dialogues** per weight update
- Without this, batch of 2 gives noisy, unstable gradients

### Mixed Precision (AMP)
- `use_amp = True` → FP16 forward pass, FP32 gradient accumulation
- Cuts VRAM usage by ~2× — critical for fitting RoBERTa + 3 encoders on 4GB GPU

### Early Stopping + Top-K Checkpoints
- If val WF1 does not improve for `patience = 25` epochs → training stops
- Top `top_k = 5` checkpoints by val WF1 are saved to `checkpoints/`
- At end of training, their predictions are **averaged** (ensemble) for final evaluation

## Loss Function Design

Total loss = **three components**:

```
L_total = L_CE  +  0.3 × L_shift  +  0.1 × L_conf_entropy
```

### 1. CrossEntropyLoss + Class Weights (`L_CE`)
The main classification loss. Each class gets a weight inversely proportional to its frequency:

```
class_weights = [3.0, 10.0, 18.0, 8.0, 3.0, 8.0, 5.0]
                 neu   sur   fea   sad  joy  dis   ang
```

- Fear gets **18.0** (highest) — only 50 test samples — model must be heavily penalised for missing it
- Neutral gets **3.0** (lowest) — 1256 test samples — model already predicts it well

### 2. Label Smoothing = 0.05
Instead of forcing the model to output 100% confidence on the true class, 5% of probability
is spread across all 7 classes. This **prevents overconfidence** and stops the model from
being too certain that ambiguous utterances are neutral.

### 3. Shift Loss (`L_shift`) — weight 0.3
An auxiliary task: predict whether the emotion *changed* from the previous utterance (binary).
This forces the model to track **emotion flow**, not just classify each utterance in isolation.

### 4. Confidence Entropy Regularisation — weight 0.1
Prevents the confidence gates from collapsing (e.g., always zeroing out the audio gate).
Forces all 3 modalities to stay active and contribute throughout training.

## Key Parameters Explained

| Parameter | Value | Simple Explanation |
|-----------|-------|--------------------|
| `bert_model` | roberta-base | The language model used to understand text |
| `text_ctx` | 5-turn-pair-encoding | Reads current + 4 prior turns together as one input |
| `max_text_len` | 192 | Max words (tokens) RoBERTa reads per dialogue window |
| `class_weights` | [3,10,18,8,3,8,5] | Rare emotion = higher penalty for getting it wrong |
| `label_smoothing` | 0.05 | Prevents model from being overconfident |
| `loss` | CrossEntropyLoss | Standard classification loss — "how wrong is the prediction?" |
| `lr_pretrained` | 2e-5 | Very slow learning for RoBERTa (already trained, don't break it) |
| `lr_fusion` | 1e-4 | Faster learning for the new fusion layers (starting from scratch) |
| `batch_size` | 2 | 2 full dialogues per GPU step (VRAM limited) |
| `grad_accum` | 8 | Accumulate 8 steps → effective batch of 16 |
| `patience` | 25 | Stop if no improvement for 25 consecutive epochs |
| `top_k` | 5 | Save 5 best checkpoints and ensemble them at inference |
| `n_heads` | 8 | 8 attention heads in cross-modal attention |
| `n_layers` | 2 | 2 layers of cross-modal attention |
| `dropout` | 0.3 | 30% of neurons randomly turned off during training (prevents memorisation) |

## All Versions — V1 to V12

Each version introduced a specific improvement. All tracked in MLflow under experiment
`DMC-Fusion-MELD-V12`.

| Version | Key Change | Val WF1 | Notes |
|---------|-----------|---------|-------|
| **V1** | Baseline — text only (BERT) | ~0.38 | No audio, no visual |
| **V2** | Added audio (wav2vec) | ~0.44 | Simple concatenation fusion |
| **V3** | Added visual (ResNet) | ~0.46 | Late fusion (concat after encoding) |
| **V4** | Replaced ResNet with EfficientNet-B4 | ~0.48 | Better visual features |
| **V5** | Added GRU context encoders | ~0.51 | First temporal modelling |
| **V6** | Replaced wav2vec with WavLM | ~0.53 | Better audio representations |
| **V7** | Added cross-modal attention | ~0.55 | Modalities now interact |
| **V8** | Added speaker embeddings | ~0.56 | Model learns who is speaking |
| **V9** | Added emotion shift auxiliary loss | ~0.57 | Temporal dynamics signal |
| **V10** | 3-turn context encoding | ~0.58 | Context window experiment |
| **V11** | Focal Loss + 5-turn context | ~0.59 | Fear/surprise boost but joy collapsed |
| **V12** | CE + recall-tuned class weights + label smoothing + 5-turn | **0.625** | Best overall WF1, stable across all classes |

### Why V12 reverted Focal Loss → CrossEntropy
Focal Loss (V11) added `(1-p)^γ` to focus on hard examples. While it boosted fear slightly,
it caused **joy and neutral to collapse** — a bad trade since joy has 6× more samples than fear.
V12 uses CrossEntropy with explicit class weights instead — giving full manual control per class.

## How to Run

### Prerequisites
```bash
conda create -n dmc_fusion python=3.10
conda activate dmc_fusion
pip install torch torchvision torchaudio transformers timm mlflow scikit-learn tqdm matplotlib
```

### Step 0 — Extract features (run only once)
```bash
python src/extract_text.py
# Extracts 5-turn RoBERTa features → features/text/
# Takes ~2-3 hours. Do NOT delete features/ after this.
```

### Step 1 — Train
```bash
python src/train.py
# Trains for up to 50 epochs with early stopping (patience=25)
# Saves: best_model.pt, ensemble_config.pt, checkpoints/*.pt
# Logs everything to MLflow
```

### Step 2 — Resume after crash
```bash
python src/train.py --resume
# Automatically finds the latest checkpoint in checkpoints/
# Restores model weights, best_f1, no_improve count
# Continues from where it left off
```

### Step 3 — Evaluate
```bash
python src/evaluate.py
# Uses ensemble_config.pt if present, else best_model.pt
# Runs post-hoc calibration on val set
# Prints: classification report, WF1, Macro F1, Accuracy
# Saves: confusion_matrix_calibrated.png, per_class_f1.png
```

### Step 4 — View MLflow Dashboard
```bash
mlflow ui
# Open http://127.0.0.1:5000
# Compare runs, view metric curves, download artifacts
```

### Clean restart
```bash
del best_model.pt
del ensemble_config.pt
del checkpoints\*.pt
python src/train.py
```
> ⚠️ Never delete `features/` — text feature extraction takes 2-3 hours.

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| `CUDA error: unknown error` at mid-epoch | GPU thermal throttle / power spike | Restart PC, run `python src/train.py --resume` |
| `OOM` (out of memory) | Batch too large for 4GB GPU | Already set `batch_size=2`, `grad_accum=8` — do not increase |
| Val WF1 stuck at 0.25 for 5 epochs | Expected — RoBERTa still frozen | Normal; big jump happens at epoch 7–8 after unfreeze |
| `best_model.pt not found` | Training never completed an epoch | Check CUDA setup, run at least 1 full epoch |
| Disgust F1 = 0.0 | Only 68 val samples — too few for stable learning | Expected limitation; class weight=8 forces some detection |
| `ensemble_config.pt` paths broken after move | Absolute paths stored in file | Run `evaluate.py` with only `best_model.pt` present |
